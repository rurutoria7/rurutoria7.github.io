【模板】連通分量

Posted on 2021-09-25 Edited on 2021-09-28 In 競程

名詞解釋

  • 連通分量：數量最多、擴展範圍最大

  • 割點：把這個點拔掉會增加連通塊數量

  • 橋：把這個邊拔掉會增加連通塊數量

  • 無向連通分量

    • 橋連通分量（邊雙連通分量）：一個連通分量內，拔掉任一點連通塊數量不會增加
    • 點雙連通分量：一個連通分量內，拔掉任一邊連通塊數量不會增加

  • 有向圖連通分量

    • 強連通分量：一個連通分量內，a 能到 b、b 能到 a

DFS Tree

把一張圖 DFS，邊可以分成四種：

  • Tree Edge：DFS 過程中經過的邊。DFS 時，走向沒走過的臨點的鄰邊都是 Tree Edge。
  • Back Edge：通向祖先的鄰邊。
  • Forward Edge：不再 DFS Tree 上，通向子樹的鄰邊。
  • Cross Edge：通向不存在父子關係的點的鄰邊。

在 DFS 時，遇到走訪過的邊就是下面三種之一。 並且無向圖只存在 Tree Edge 和 Back Edge（Forward Edge 因為無向等同於 Back Edge；u 若存在 Cross Edge 連到 v，那在 v 時就應改把這條邊當成 Tree Edge）。

Tarjan's Algorithm

\(low[u]\)

\(low[u]\) 代表從 u 出發，經過子樹，再經過最多一條 Back Edge 能達到的最淺深度。

維護 \(low\) 很簡單，當在 DFS 時從 \(u\) 走到 \(v\)，求 \(low[u]\)：

  • 走到 Tree Edge，直接拿他的 \(low\) ：\(low[u] = min(low[u], low[v])\)
  • 走到 Back Edge（除了從走來的這條以外），拿他的深度：\(low[u] = min(low[u], dep[v])\)
  • 走到 Forward Edge 和 Cross Edge 無視。

找割點

若點 \(u\) 是割點，代表它可以將至少一個子節點 \(v\) 和父節點隔開（拿掉 \(u\) 會導致子節點走不到父節點）。 若且唯若 \(low[v] \leq dep[u]\)
